## 摘要
真实医疗操作端到端自动化对当前基准测试中代表性不足的三种能力提出了挑战：政策密度——决策必须基于庞大的医疗、保险和操作规则库；多角色组合——单个任务要求代理扮演多个角色并进行交接；多边交互——中间工作流程步骤是多轮对话，如同行评审和患者外联。我们引入了χ-Bench，这是一个跨三个领域（提供者预先授权、支付者利用管理和护理管理）的长周期医疗工作流程基准测试。每个任务在一个由87个MCP工具暴露的20个医疗应用的高保真模拟器中向代理提供一个临床案例，代理必须通过工具调用和编写角色工件（遵循1290+文档管理式医疗操作手册技能）将其驱动至终端状态。在30个代理框架/模型配置中，最佳代理仅解决28.0%的任务，没有代理在严格的pass^3上超过20%，而在单个会话中执行所有任务则使性能降至3.8%。这些结果提出了一个假设：在其他政策密集、角色组合、不可逆的企业领域，类似的差距很可能出现。

## Abstract
End-to-end automation of realistic healthcare operations stresses three capabilities underrepresented in current benchmarks: policy density, decisions must be grounded in a large library of medical, insurance, and operational rules; Multi-role composition: a single task requires the agent to play multiple roles with handoffs; and multilateral interaction: intermediate workflow steps are multi-turn dialogs, such as peer-to-peer review and patient outreach. We introduce $χ$-Bench, a benchmark of long-horizon healthcare workflows across three domains: provider prior authorization, payer utilization management, and care management. Each task hands the agent a clinical case in a high-fidelity simulator of 20 healthcare apps exposed via 87 MCP tools, which it must drive to a terminal status through tool calls and writing the role's artifacts, guided by a 1,290+ document managed-care operations handbook skill. Across 30 agent harness/models configurations, the best agent resolves only 28.0% of tasks, no agent clears 20% on strict pass^3, and executing all tasks in a single session slumps the performance to 3.8%. These results raise the hypothesis that similar gaps are likely to surface in other policy-dense, role-composed, irreversible enterprise domains.

- **现实痛点**：美国医疗系统行政负担沉重，尤其是**先授权（Prior Authorization）**、**利用管理（Utilization Management）** 和**护理管理（Care Management）** 三个领域工作流程繁琐、手工操作多，极易造成延误和资源浪费。

## 2. 论文提出的方法论

### 核心思想

### 关键技术细节

- **χ-World Engine**：本地化的高保真模拟器，包含20个日常医疗应用，暴露为**151个REST API**和**87个MCP工具**（分布于3个MCP服务器），拥有约5000条患者记录和约90个医疗工作者。

- **任务定义**：形式化为分层部分可观察马尔可夫决策过程（POMDP）。每个任务包含指令、容器化环境、角色限定的工具界面、以及双层验证器。交接是不可逆的：一旦提交或路由，不能编辑或重跑。
- **任务构建与组成**：
    1. **案例生成**：采样终端世界状态，再由LLM（Claude Opus 4.7）结合结构化JSON采样生成上游工件、评分标准。
    2. **人工走查**：标注员在真实UI上完整走完流程，记录轨迹、数据库状态、交接记录作为真值。
    3. **多评审者审查**：至少1名执业医疗工作者和5名作者审核临床精度，识别PHI残留，进行临床合理性检查。最终筛选出**75个代表性长周期任务**（人工平均21步，最多40步）。

## 6. 论文的主要结论与发现

- **整体性能低下**：最佳配置Claude Code+Claude Opus 4.6仅达到**28.0% pass@1**；严格pass^3指标下没有代理超过20%（最高18.7%）。
- **马拉松性能灾难性降至3.8%**，多任务连续场景下几乎失效。
- **端到端双代理竞技场**中，Pass@1从30.4%直接降至**0%**，表明跨角色协作是巨大瓶颈。
- **失败模式主导因素**：
    - **临床推理错误（35.4%）**：正确看到了证据但做出错误医疗或协议判断。
    - **工作流未完成（23.3%）**：从未调用所需的终端动作。
    - **政策合规错误（13.2%）**：文字误读政策标准。
    - **放弃/卡住（15.6%）**：时间耗尽、无限循环、过早关闭、拒绝执行。
    - **工具使用错误（10.7%）**：主要出现在DeepAgents框架中。
    - **幻觉（0.8%）** 虽少但存在。
- **技能库效果因领域而异**：UM高度依赖手册（移除后性能腰斩），PA反而在无手册时略好（因有手册时过度验证/拒绝提交），CM始终很低（难度在对话驱动）。
- **MCP与CLI对比**：总体中性至略差，表明工具界面格式对这类任务并非关键瓶颈。

## 7. 优点

1. **高保真度仿真**：模拟了真实的医疗软件环境、不可逆交接、多轮对话、状态机约束，远超现有单纯问答或短链基准。
2. **大规模技能库**：1279个文档的手册结构合理（渐进式披露、角色路由、共享附录），首次在如此规模下评测代理的技能利用能力。
3. **全面多维评估**：除了pass@1，还引入pass^3（可靠性）、马拉松（持久性）、E2E竞技场（跨角色协作），以及详细的失败模式分类。
4. **严格的验证流程**：确定性检查+LLM评判器双重机制，人工走查与多位专家审核确保真值质量。
5. **开源与可复现**：提供了代码、环境、数据、评测工具链（GitHub），社区可重复验证。
6. **失败模式洞察**：不是仅报告最终分数，还深入分析了错误类型，为未来改进提供了明确方向（如临床推理、政策解读、对话伦理）。

## 8. 不足与局限

1. **仅限语言模态**：真实医疗工作常需要处理医学影像（X光、病理切片）和语音交流（电话沟通），当前基准只支持文本命令，缺乏多模态能力。
2. **任务覆盖有限**：虽然覆盖了高影响力的三个领域，但医疗行业有数百种长尾工作流，外推能力有待验证。
3. **评判模型单一**：只使用Claude Opus 4.7作为LLM评判器，不同评判模型对结果的影响未研究，可能存在系统性偏差。
4. **技能副作用**：大型手册在PA领域引发了过度验证/拒绝行为，导致性能反而下降，说明技能设计本身也可能带来负面影响。
5. **时间限制**：单次任务固定1800秒超时，可能不利于需要深思熟虑的复杂推理（但现实中也存在时间压力）。
6. **无法完全模拟人性化互动**：例如CM中护理经理与患者的沟通，模拟的“拒绝”“犹豫”仍与真实人类反应有差距，可能低估或高估代理的实际对话能力。
7. **框架差异未深入分析**：虽然比较了多个框架，但未深入分析框架设计（如错误恢复、上下文管理）对失败模式的具体影响。
