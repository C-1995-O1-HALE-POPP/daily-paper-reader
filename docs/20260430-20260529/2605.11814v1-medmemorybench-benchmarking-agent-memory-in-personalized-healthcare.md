## 摘要
大规模部署个性化医疗智能体需要极其精确、安全且能够长期临床追踪的记忆机制。然而，现有基准测试主要关注日常开放域对话，未能捕捉真实世界医疗应用中高风险的高复杂性。受服务于数千万活跃用户的行业领先健康管理智能体的严格生产需求驱动，我们提出了MedMemoryBench。我们开发了一个人机协作流水线，基于临床验证的合成患者原型，合成高度真实、长时程的医疗轨迹。该过程产生了一个大规模、经专家验证的数据集，包含约2000个会话和16000轮交互。关键的是，MedMemoryBench摒弃了传统的静态评估，开创了一种“边构建边评估”的流式评估协议，精确模拟生产环境中的动态记忆积累。此外，我们形式化并系统研究了记忆饱和这一关键现象，其中持续的信息涌入会主动降低检索和推理的鲁棒性。全面的基准测试揭示了主流架构中的严重瓶颈，特别是在复杂医疗推理和噪声鲁棒性方面。通过暴露这些根本缺陷，MedMemoryBench为开发稳健、可投入生产的医疗智能体奠定了重要基础。

## Abstract
The large-scale deployment of personalized healthcare agents demands memory mechanisms that are exceptionally precise, safe, and capable of long-term clinical tracking. However, existing benchmarks primarily focus on daily open-domain conversations, failing to capture the high-stakes complexity of real-world medical applications. Motivated by the stringent production requirements of an industry-leading health management agent serving tens of millions of active users, we introduce MedMemoryBench. We develop a human-agent collaborative pipeline to synthesize highly realistic, long-horizon medical trajectories based on clinically grounded, synthetic patient archetypes. This process yields a massive, expertly validated dataset comprising approximately 2,000 sessions and 16,000 interaction turns. Crucially, MedMemoryBench departs from traditional static evaluations by pioneering an "evaluate-while-constructing" streaming assessment protocol, which precisely mirrors dynamic memory accumulation in production environments. Furthermore, we formalize and systematically investigate the critical phenomenon of memory saturation, where sustained information influx actively degrades retrieval and reasoning robustness. Comprehensive benchmarking reveals severe bottlenecks in mainstream architectures, particularly concerning complex medical reasoning and noise resilience. By exposing these fundamental flaws, MedMemoryBench establishes a vital foundation for developing robust, production-ready medical agents.

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：构建反映真实医疗部署的高保真数据，并采用“边构建边评估”的流式协议，模拟在线记忆积累过程。同时，引入噪声增强和陷阱事件，评估记忆饱和下的鲁棒性。
- **关键技术细节（四阶段数据构建流水线）**：
  1. **患者档案构建**：基于去标识真实病例，创建20个代表性慢性病（糖尿病、心血管、呼吸系统等）患者原型，并扩展生活方式、家庭背景等外围属性。
  2. **疾病进展事件生成**：为每个患者生成经专家验证的一年诊断轨迹报告，包括症状、转折点、并发症、治疗和康复。分解为事件并组织成时间-因果有向图，并在图中注入陷阱事件（如过敏史、药物禁忌、经济约束）。
  3. **多轮会话模拟**：实例化患者智能体和医生智能体，基于事件图进行交互。患者按时间线发起会话，医生基于历史记忆回应。每轮会话后记录摘要并反馈到后续会话，保持长期连贯性。
  4. **记忆提取与查询构建**：使用多智能体框架（Trap Agent、Query Agent、Critique Agent）从会话记忆中提取高优先级事实，构建六类查询。医学专家验证轨迹、对话和问答对，确保临床合理性。
- **噪声增强**：引入两种干扰：① 通用健康知识咨询（38个预设主题）；② 代家属咨询（每个患者关联2-4个虚拟家庭成员）。噪声会话随机插入原始会话序列，标记为 `is_noise:true`。
- **流式评估协议**：
  - 方法按严格时间顺序接收会话，每 `Δ=10` 个会话设置一个检查点。
  - 每个检查点评估时，只允许使用截至该点的记忆（防止未来信息泄漏）。
  - 最终成绩由所有检查点的结果聚合。
- **查询类型**：六类覆盖精确事实检索（EEM、TLA）、时间冲突解决（SUA）和医疗知识推理（MQ、IG、MCD）。其中 MCD 评估节点覆盖率、因果正确性和链条完整性。

## 6. 主要结论与发现

1. **所有现有方法远未达到令人满意的水平**，尤其在推理密集型任务（IG、MCD）上表现很差。
2. **中等粒度记忆效果最佳**：如 A-Mem、MemOS、Letta 在 Efficient 设置下总体最好；过细（Mem0）或过粗（MIRIX）均不利。图结构方法因难以处理高重叠、强时间依赖的医疗记忆而落后。
3. **检索是主要瓶颈**：Letta 的优势部分来自其 Core Memory 机制（关键信息始终在系统提示中），减少对精确检索的依赖。其他方法记忆库虽大但检索不准确。
4. **记忆饱和严重损害推理鲁棒性**：Mixed 设置下所有方法性能下降，尤其 IG 和 MCD。冗余噪声不仅增加检索难度，还干扰记忆构建本身。
5. **高召回并不直接转化为高回答质量**：幻觉率普遍偏高（>70%），表明模型在证据不足时倾向于编造而非拒绝。
6. **成本效率失衡**：大部分方法的构建阶段成本是查询阶段的数倍，但高性能方法（如 Letta）的成本并不极端，暗示存在更优的效率-性能平衡点。
