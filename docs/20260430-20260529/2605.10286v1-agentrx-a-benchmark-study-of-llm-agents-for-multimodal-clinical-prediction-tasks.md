## 摘要
构建有效的临床决策支持系统需要综合处理复杂的异构多模态数据。这些模态包括时序电子健康记录数据、医学影像、放射学报告和临床笔记。基于大语言模型的智能体在各种医疗任务中表现出色，尤其是那些涉及文本模态的任务。考虑到医疗数据在院际系统间的碎片化问题，协作式智能体框架为缓解数据共享挑战提供了有前景的方向。然而，LLM智能体在多模态临床风险预测中的有效性仍缺乏系统研究。本研究基于大规模真实世界数据，对基于LLM的智能体在临床预测任务中的表现进行了系统评估。我们评估了单模态和多模态场景下的性能，并量化了单智能体系统与多智能体系统之间的性能差距。研究结果突显了单智能体框架优于朴素的多智能体系统，能更好地处理多模态数据，且校准更佳。这强调了改进多智能体协作以更好处理异构输入的迫切需求。通过开源我们的代码和评估框架，本研究为医疗领域智能体系统的未来发展提供了新基准。

## Abstract
Building effective clinical decision support systems requires the synthesis of complex heterogeneous multimodal data. Such modalities include temporal electronic health records data, medical images, radiology reports, and clinical notes. Large language model (LLM)-based agents have shown impressive performance in various healthcare tasks, especially those involving textual modalities. Considering the fragmentation of healthcare data across hospital systems, collaborative agent frameworks present a promising direction to mitigate data sharing challenges. However, the effectiveness of LLM agents for multimodal clinical risk prediction remains largely unexamined. In this work, we conduct a systematic evaluation of LLM-based agents for clinical prediction tasks using large-scale real-world data. We assess performance in unimodal and multimodal settings and quantify performance gaps between single agent and multi-agent systems. Our findings highlight that single agent frameworks outperform naive multi-agent systems, are better at handling multimodal data, and are better calibrated. This underscores a critical need for improving multi-agent collaboration to better handle heterogeneous inputs. By open-sourcing our code and evaluation framework, this work offers a new benchmark to support future developments relating to agentic systems in healthcare.

## 2. 论文提出的方法论：核心思想、关键技术细节
- **核心思想**：构建并对比三种 agent 架构：
  1. **单智能体单模态**：仅使用患者摘要（PS）文本进行预测。
  2. **单智能体多模态**：将所有可用模态（PS、EHR、CXR、RR）合并到同一个上下文窗口内，由单一 agent 处理。
  3. **多智能体多模态**：每个模态独立由一个专用 agent 处理，然后通过多种协作机制（如投票、辩论、元提示、轨迹链+多模态法官等）综合决策。
- **关键技术细节**：
  - **数据预处理**：
    - EHR 时间序列（48 小时内）被序列化为 `[T0+Δt] Variable=Value` 文本，高频流截断至 500 步（前 100 + 后 400）。
    - CXR 图像选取 ICU 前 48 小时内的最新 AP 视图。
    - RR 报告拼接为单一文本。
    - PS 从出院笔记中提取入院前信息，避免数据泄露。
  - **Agent 推理**：所有 agent 基于特定系统提示进行推理，输出概率或分类。
  - **多智能体协作策略**（共 6 种）：
    - Majority Vote：各 agent 独立输出，取平均。
    - Debate：多轮辩论直至共识。
    - Meta-Prompting：元 agent 决定是否召集团队。
    - Traj-CoA + Multimodal Judge：分块处理 EHR，再由多模态 judge 综合。
    - MDAgents / MedAgents：预定义专家角色协作。
- **公式/算法**（文字描述）：
  - 单智能体：构造提示 `P_task` + 模态数据 → 调用 LLM 得到预测。
  - 多智能体：对每个模态 m，构造 `P_task` + `x_m` → 得到概率 `p_m` → 对 `p_m` 平均或通过特定协议得到最终预测。

## 3. 实验设计：数据集、基准、对比方法
- **数据集**：来自 MIMIC-IV（EHR）、MIMIC-CXR（图像）、MIMIC-IV-Note（放射报告和患者摘要）。共约 17,773 训练/4,925 测试（死亡率任务），17,476/4,845（住院时长任务）。要求所有样本必有 PS，其他模态按需配对。
- **预测任务**：

  - **长住院时间**（住院 ≥ 7 天）
- **对比基线**：

  - **单智能体基线**：Zero-shot、Few-shot（1正1负）、Chain-of-Thought、Self-Consistency+CoT、Self-Refinement
  - **多智能体基线**：Majority Vote、Debate、Meta-Prompting、Traj-CoA、MDAgents、MedAgents
- **模型骨干**（均为 7-8B 参数）：

## 6. 论文的主要结论与发现
1. **单智能体多模态显著优于多智能体**：在所有骨干和任务上，单 agent 零样本多模态胜过多数多 agent 协作框架（如 Majority Vote 和 Debate）。例如，Qwen 单 agent 死亡率 AUROC 0.756，而 Debate 仅 0.631。
2. **多模态带来收益**：与单模态（仅 PS）相比，单 agent 加入更多模态后 AUROC 和校准均改善。例如 Qwen 死亡率 AUROC 从 0.667 提升至 0.756，ECE 从 0.025 降至 0.023。
3. **多智能体系统校准退化**：随着模态增加，多 agent 投票系统的 ECE 恶化（如 Qwen 从 0.068 升至 0.111），表明聚合机制引入了过confidence。
4. **专家模型在文本任务上可超越监督基线**：Huatuo 死亡率 AUROC 达 0.700（Few-shot），高于 BioBERT 的 0.680。
5. **多智能体协作存在“回声室”和过早收敛**：Qwen 在所有样本的首轮即达成共识（无实质辩论），LLaVaMed 有 52.6% 样本从未达成共识，表明当前多 agent 框架难以有效融合异构信息。
6. **Traj-CoA 表现最好**：它保留了多模态 judge 的全局上下文访问，减轻了信息丢失。
