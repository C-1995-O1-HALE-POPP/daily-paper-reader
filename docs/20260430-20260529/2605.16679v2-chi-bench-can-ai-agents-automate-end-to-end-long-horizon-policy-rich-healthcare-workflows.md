## 摘要
真实医疗运营的端到端自动化对三种能力提出了要求，而这些能力在当前基准测试中代表性不足：政策密集性——决策必须基于庞大的医学、保险和操作规则库；多角色组合——单个任务要求智能体扮演多个角色并进行交接；以及多边交互——中间工作流步骤涉及多轮对话，如同行评审和患者外展。我们引入了χ-Bench，这是一个跨三个领域（提供者预先授权、支付者利用率管理和护理管理）的长周期医疗工作流基准。每个任务将临床案例交给一个高保真模拟器，该模拟器通过87个MCP工具暴露了20个医疗应用，智能体必须通过工具调用和撰写角色工件，在1290+文档的管理式医疗操作手册技能的指导下，将工作流驱动至终端状态。在30种智能体配置/模型组合中，最佳智能体仅解决了28.0%的任务，没有智能体在严格的pass^3指标上超过20%，并且在一个会话中执行所有任务时性能骤降至3.8%。这些结果提出了一个假设：类似差距很可能在其他政策密集、角色组合、不可逆的企业领域中出现。

## Abstract
End-to-end automation of realistic healthcare operations stresses three capabilities underrepresented in current benchmarks: policy density, decisions must be grounded in a large library of medical, insurance, and operational rules; Multi-role composition: a single task requires the agent to play multiple roles with handoffs; and multilateral interaction: intermediate workflow steps are multi-turn dialogs, such as peer-to-peer review and patient outreach. We introduce $χ$-Bench, a benchmark of long-horizon healthcare workflows across three domains: provider prior authorization, payer utilization management, and care management. Each task hands the agent a clinical case in a high-fidelity simulator of 20 healthcare apps exposed via 87 MCP tools, which it must drive to a terminal status through tool calls and writing the role's artifacts, guided by a 1,290+ document managed-care operations handbook skill. Across 30 agent harness/models configurations, the best agent resolves only 28.0% of tasks, no agent clears 20% on strict pass^3, and executing all tasks in a single session slumps the performance to 3.8%. These results raise the hypothesis that similar gaps are likely to surface in other policy-dense, role-composed, irreversible enterprise domains.

  - 马拉松会话：所有25任务串联时，pass@1骤降（PA从29.3%降至8.0%，UM从32.0%降至2.7%，CM从1.3%降至0%）。
- **失败模式分析**：对5886次失败试验分类，主要失败来源：
  - 临床推理（35.4%）：包括标准误用、非法同意（CM领域）。
  - 工作流完成（23.3%）：跳过必需步骤、提前关闭。
  - 政策合规（13.2%）：标准解读错误。
  - 放弃或卡住（15.6%）：超时、循环、拒绝行动。
  - 工具使用错误（10.7%）：主要集中在DeepAgents框架的致命工具调用。
  - 幻觉（0.8%）：制造事实。
- **两代理对决**：PA领域端到端提供者+支付者双代理模式导致pass@1从30.4%降至0%。
- **手册技能的影响**：UM领域强依赖于手册（去掉领域手册使pass@1从32%降至17.3%）；PA领域手册反而导致过度验证和拒绝提交；CM领域手册效果微弱（因为主要挑战是对话而非政策）。
- **结论**：当前AI代理在策略密集、多角色、不可逆的企业工作流中表现严重不足，类似缺口很可能存在于其他领域。

## 7. 优点：方法或实验设计上的亮点

- **高度还原真实临床环境**：χ-World Engine模拟了20个医疗应用、87个MCP工具、5000+患者活动，且工作流状态机有29种状态和明确的法律转换，贴合真实操作。
- **系统化的任务构建**：从案例生成到人工走查再到多重审阅，确保临床准确性和无泄漏，且每个预期动作都能追溯到政策章节。
- **双层验证器**：确定性检查+LLM法官的结合，既防止了纯粹的模式匹配，又避免了LLM法官的完全主观性。
- **全面的评估指标**：不仅报告pass@1，还有pass@3和pass^3（严格可靠性），以及成本、步骤数等效率指标，提供了多维度视角。
- **详细的失败分析**：对近6000次失败试验进行两层分类，并量化每种模式的比例，为改进方向提供具体指导。
- **大规模对比**：覆盖30种配置，包括主流闭源模型和多种开源框架，结果具有广泛参考价值。

## 8. 不足与局限

- **仅限语言代理**：未考虑多模态推理（如医学影像解读、语音交互），而真实医疗操作常需这些能力。
- **领域覆盖有限**：只包含PA、UM、CM三个领域，医疗系统中有数百种长尾工作流，可能不具全面代表性。
- **法官模型单一**：仅使用Claude Opus 4.7作为LLM法官，可能存在同族偏见；计划中的跨法官敏感性检查尚未完成。
- **任务数量较少**：目前仅75个任务，每个领域25个，可能不足以覆盖所有临床场景和难度分布。
- **模拟与现实差距**：模拟环境中的患者、支付者行为可能简化了真实世界的复杂性（如患者不配合、支付者政策频繁变更）。
- **静态基准**：手册和政策是固定的，而真实世界中政策会随时间变化，基准未能测试代理适应变化的能力。
- **成本高**：大规模评估（6750试验）计算成本较高，可能限制其他研究者的复现或扩展。
- **未考虑监管与安全**：论文本身作为压力测试，警告低通过率代理部署于实际患者护理有风险，但未提出具体安全约束或人类介入机制。
