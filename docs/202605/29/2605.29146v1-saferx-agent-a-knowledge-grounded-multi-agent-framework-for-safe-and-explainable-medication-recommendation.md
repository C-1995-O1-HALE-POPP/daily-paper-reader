## 摘要
药物推荐预测患者就诊时的用药方案，但现有方法仍面临两个关键挑战。在模型层面，传统药物推荐方法仅预测结构化的药物代码，缺乏充分的证据支撑，而LLM智能体可以利用更丰富的临床上下文，但可能缺乏安全性验证和可追溯性。在任务层面，现有基准测试通常使用宽泛的药物类别，忽略了亚组级别的安全性差异，可能导致风险高估。我们引入了首个基于第四级ATC代码生成的细粒度药物推荐设置。我们提出了安全处方智能体（SafeRx-Agent），这是一个基于知识的多智能体框架，利用患者上下文、外部临床知识和安全性验证来推荐可追溯的药物集合。在MIMIC-III和MIMIC-IV数据集上的实验结果表明，SafeRx-Agent在控制药物相互作用、禁忌症和药物集合大小的同时，提高了细粒度药物预测的准确性。

## Abstract
Medication recommendation predicts medications for patient visits, but existing methods still face two key challenges. At the model level, traditional drug recommendation methods only predict structured drug codes with limited evidence grounding, while LLM agents can use richer clinical context but may lack safety verification and traceability. At the task level, existing benchmarks often use broad medication categories, which ignore subgroup-level safety differences and can lead to risk overestimation. We introduce the first fine-grained medication recommendation setting based on fourth-level ATC code generation. We propose Safe Prescription Agent (SafeRx-Agent), a knowledge-grounded multi-agent framework that uses patient context, external clinical knowledge, and safety verification to recommend traceable medication sets. Experimental results on MIMIC-III and MIMIC-IV datasets show that SafeRx-Agent improves fine-grained medication prediction accuracy while controlling drug interactions, contraindications, and medication set size.

## 2. 方法论

### 核心思想
SafeRx-Agent 是一个知识驱动的多智能体框架，包含三个主要阶段：**路由 → 多智能体生成 → 安全验证**。它利用外部临床知识库（ICD/CCS 分类、ATC 分类、MEDI 药物适应证、TWOSIDES 药物相互作用、openFDA 禁忌症）来约束生成并验证安全性。

### 关键组件与流程（Algorithm 1）
1. **专家面板（Expert Panel）**：预构建 \(K\) 个专科专家（如心血管、内分泌、呼吸等）和一个通用支持专家（E_univ）。专家通过聚类患者嵌入和 ICD 章节映射生成。
2. **多智能体生成**：
   - **ROUTE**：根据患者当前和既往 ICD 代码，稀疏激活相关专家（加权评分），E_univ 始终激活。
   - **SUMMARIZE**：每个激活专家提取专科相关的摘要（活跃问题、器官功能障碍、用药史等）和就诊理由（ρ_i）。
   - **GENERATE**：基于摘要和 MEDI 检索的适应证候选药物，生成 ATC-L4 候选集（含置信度）。
   - **CRITIQUE**：合并所有专家候选集，去除重复和缺乏患者级别支持的药物，输出候选集 \(\tilde{M}\)。
3. **安全验证**：
   - **FIND FLAGS**：利用 DDI 矩阵和禁忌症矩阵检测候选集中的安全标志（f = (drug, relation, evidence)）。
   - **VERIFY**：根据患者上下文（如既往是否耐受）逐项裁定保留或移除，优先保留既往用药，移除无理由的高风险药物。

### 算法流程（文字描述）
1. 根据患者记录激活专家子集 A。
2. 每个专家生成摘要和候选药物。

4. 对初选集进行 DDI 和禁忌症扫描，对每个标志进行上下文裁决，输出最终集 \(\hat{M}_T\)。

1. **预测准确性提升**：SafeRx-Agent 在 MIMIC-IV 上使用 GPT-5.2 时达到 F1=0.3934（带过滤），优于所有直接提示和 Agent 基线；在 MIMIC-III 上也取得最佳或次佳性能。无过滤变体在多个模型上获得最高 F1（如 Gemma3-27B-IT 上 F1=0.3882）。

3. **细粒度粒度的重要性**：ATC-L4 下 DDI-B 和 Contra-B 明显低于 ATC-L3（如 MIMIC-IV 上 30.13% vs 63.98%），证明粗粒度会高估风险。
4. **组件贡献**：消融实验显示移除专科专家导致 F1 最大下降（-0.057），移除医疗资源导致召回大幅下降（-0.073），批判主要针对单专家提出的假阳性（96.2% 被移除的药物仅由一个专家提出）。每个专家均贡献正向 F1 提升（+0.101~+0.232）。

## 7. 优点

- **首次提出细粒度 ATC-L4 药物推荐任务**，纠正了粗粒度评估的风险高估问题，更贴近临床实际。
- **知识推理与生成结合**：利用 ICD/ATC 分类、MEDI 适应证、DDI/禁忌症矩阵，将外部知识显式注入生成和验证，提升可靠性和可解释性。
- **多智能体协同设计**：专科路由避免无关专家干扰，批判环节有效去除弱证据候选，安全验证通过矩阵检索+上下文裁决实现精细控制。
- **完备的评估体系**：同时评估预测精度和多种安全指标（DDI 二元/加权、禁忌症），并分析了粒度影响、效率、专家贡献。
- **模块化可扩展**：专家面板和知识资源可替换，适用于不同医疗场景。

## 8. 不足与局限

- **外部知识依赖**：安全验证依赖 TWOSIDES 和 openFDA 的数据覆盖度，未包含过敏、剂量错误、特殊人群（肝肾损伤）等风险；DDI 和禁忌症矩阵基于 ATC-L4 映射，可能丢失子类细节。
- **回顾性离线评估**：仅在 MIMIC 数据集上测试，缺乏前瞻性临床验证。临床部署需要医生监督和实时验证。
- **计算开销与可扩展性**：相比直接提示，SafeRx-Agent 增加约 3 倍 Token 消耗和 2 倍延迟，对实时要求高的场景可能受限于资源。
- **假阴性问题**：4 个假阴性药物未被任何专家提出（如抗真菌、抗抑郁药），表明框架可能遗漏基于隐式推理的用药。
- **专家构建依赖数据分布**：专家聚类基于 MIMIC 数据分布，迁移到其他医院可能需重新构建。
- **未探索多轮生成或迭代修正**：当前只进行一轮生成+验证，未考虑临床反馈循环。
