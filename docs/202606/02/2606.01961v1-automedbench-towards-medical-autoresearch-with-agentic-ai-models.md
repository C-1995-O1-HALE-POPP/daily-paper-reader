## 摘要
自主代理正日益被期望支持端到端的医学AI研究工作流程，超越孤立的预测任务或短形式临床问答。然而，现有的医学代理基准主要评估最终输出，对研究过程中代理行为的可见性有限。为弥补这一差距，我们提出AutoMedBench，这是一个面向工作流的基准，用于跨多种医学成像和多模态推理任务的自主医学AI研究，将代理执行组织为统一的五阶段工作流（S1-S5）：计划、设置、验证、推理和提交。它包含长时间跨度的任务，每次运行平均33个代理轮次，涵盖五个研究轨道：分割、图像增强、视觉问答（VQA）、报告生成和病变检测。每个任务在两种难度级别下评估：精简版和标准版，它们使用相同的数据和指标，但任务简报支架的数量不同，每次运行使用最终任务性能和S1-S5阶段得分进行评分，从而实现对从初始任务简报到最终提交工件各阶段的分析。在数千次记录运行中，阶段级评分显示，验证是平均最弱的工作流阶段，而设置是最强的，表明当前代理更擅长使流程可执行，而非验证其可靠性。运行后错误分析进一步显示，验证和提交失败主导了标记错误，分别占触发代码的37.7%和38.1%，而任务理解错误很少，仅占0.9%，并且触发一个错误代码的运行的平均总分比无错误代码的运行低48%。

## Abstract
Autonomous agents are increasingly expected to support end-to-end medical-AI research workflows, moving beyond isolated prediction tasks or short-form clinical question answering. However, existing medical agent benchmarks primarily evaluate final outputs, providing limited visibility into agent behavior within the research process. To address this gap, we present AutoMedBench, a workflow-aware benchmark for autonomous medical-AI research across diverse medical imaging and multimodal inference tasks, organizing agent execution into a unified five-stage workflow (S1-S5): Plan, Setup, Validate, Inference, and Submit. It comprises long-horizon tasks with each run averaging 33 agent turns, spanning five research tracks: segmentation, image enhancement, visual question answering (VQA), report generation, and lesion detection. Each task is evaluated under two difficulty tiers, Lite and Standard, which use the same data and metrics but differ in the amount of task-brief scaffolding, and each run is scored using both final task performance and S1-S5 stage scores, enabling stage-level analysis from the initial task brief to the final submitted artifact. Across thousands of recorded runs, stage-level scoring reveals that Validate is the weakest workflow stage on average, whereas Setup is the strongest, suggesting that current agents are better at making pipelines executable than at verifying their reliability. Post-run error analysis further shows that verification and submission failures dominate tagged errors, accounting for 37.7% and 38.1% of fired codes respectively, whereas task-understanding errors are rare at 0.9%, and runs with one fired error code have a 48% lower overall score than runs with no error code on average.

## 1. 核心问题与整体含义（研究动机与背景）

- **研究动机**：现有医学AI代理基准大多只评估最终输出（如问答准确率或任务成功率），无法揭示代理在研究过程中的行为、失败原因及阶段级能力。在实际的长时间跨度的医学AI研究流程中，代理需要跨多个交互回合保持上下文，而失败可能来自不同阶段并逐级累积，最终被单一分数掩盖。因此，需要一种工作流感知的基准来评估代理的端到端研究能力。
- **整体含义**：该工作旨在推动自主医学AI研究（AutoResearch）的发展，检验当前最前沿的AI代理是否能够像人类研究者一样完成规划、环境配置、验证、推理和提交的完整研究流程，而不仅仅是回答医学问题或完成单步任务。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：提出 **AutoMedBench**，这是一个工作流感知的基准，将每个代理的医学AI研究过程组织为统一的五阶段工作流（S1-S5）：
  - S1 Plan（计划）：理解任务，选择可行方法，编写 plan.md。
  - S2 Setup（设置）：安装依赖，加载模型/API，验证路径和输出。
  - S3 Validate（验证）：运行试点案例，检查中间输出，修正管道错误。
  - S4 Inference（推理）：运行完整推理，生成预测文件。
  - S5 Submit（提交）：验证提交模式并提交最终工件。
- **关键技术细节**：
  - **双重评分体系**：每个运行获得 **Agentic Score**（基于S1-S5阶段完成度，加权：S1 25%，S2 15%，S3 35%，S4 15%，S5 10%）和 **Task Score**（基于固定保留测试的最终工件质量，如Dice、SSIM、准确率等）。总体分 = 0.5×Agentic + 0.5×Task。
  - **难度层级**：每个任务有 **Lite** 和 **Standard** 两种难度，仅任务简报的支架程度不同（Lite提供更详细的方法、依赖和验证提示），而数据、指标、提交模式完全相同。
  - **控制环境**：代理在隔离容器中运行，具有GPU和网络访问；评估容器无法访问外部网络且无法看到私有参考。强制推理-only协议（不允许训练或微调）。
  - **错误编码**：运行后基于完整交互日志，分配五种因果错误代码（E1理解错误，E2数据/模型设置错误，E3验证/恢复错误，E4实现/执行错误，E5提交错误）。错误代码仅用于诊断，不影响分数。
- **公式**：无复杂数学公式，评分规则为加权求和和确定性检查。

## 6. 主要结论与发现

- **领导者分数并不揭示统一最优画像**：总体分数差距15.3分，但不同代理在不同轨道和阶段表现各异。
- **任务质量落后于工作流完成度**：所有代理的Agentic Score高于Task Score，说明完成工作流程不等于产出高质量医学工件。
- **不同医学轨道暴露不同弱点**：检测得分最高（输出格式固定、预训练模型成熟），VQA和报告生成最困难。
- **验证是工作流中心瓶颈**：S3（验证）平均分最低，S2（设置）平均分最高，表明代理擅长搭建可执行管道但缺乏可靠性验证能力。
- **更多支架并不总是提升性能**：从Standard到Lite，部分代理提升（最多+8.3%），但GPT-5.4下降16.3%，说明支架可能引入脆弱性。
- **更高成本不保证更好性能**：成本-性能相关性弱且因轨道而异。
- **工程型失败主导代理故障**：验证错误（E3占37.7%）和提交错误（E5占38.1%）最频繁，理解错误（E1）仅0.9%。
- **出现错误代码导致性能断崖**：有一个错误代码的运行平均总分下降48%，两个以上则保持低分。
- **强代理更善于从多次错误中恢复**：顶级代理并非没有错误，而是能更有效恢复。

## 7. 优点

1. **工作流感知评估**：首次将医学AI研究过程分解为可评分的阶段，提供了比单一最终输出更丰富的诊断信息。
2. **跨任务统一框架**：覆盖五种不同类型、多种模态的医学AI任务，使用相同的五阶段流程和评分协议，使得异构任务可比较。
3. **双重难度控制**：Lite/Standard设计不仅调节难度，还允许评估代理在给支架和自主决策下的不同表现。
4. **开源基础设施**：发布完整的执行引擎和隔离沙箱，支持可重复研究。
5. **因果错误分析**：通过后运行错误编码将失败模式与性能下降定量关联，指导改进方向。

## 8. 不足与局限

- **仅限推理场景**：不支持训练或微调，而许多医学AI研究涉及模型调整。实际应用中训练能力重要。
- **代码执行自动化局限**：代理需要大量工程操作（安装包、下载权重），这与纯知识推理不同，可能使工程能力弱的模型不公平劣势。
- **未包含多智能体或检索增强**：系统故意消去了这些增强，以便研究基础模型差异，但现实应用中这些组件可能关键。
- **数据集依赖公开可用性**：部分数据集需凭证访问，实验可重复性受影响。
- **评估指标设定**：使用标准学术指标（Dice、SSIM、准确率等），但缺乏临床有效性评估（例如生成报告是否临床正确）。
- **伦理与安全**：论文未深入讨论自主医学研究代理的伦理风险（如错误诊断导致的后果）。
