## 摘要
临床AI模型在对抗性漏洞未被检测时可能使患者受到伤害，然而正式的安全审计需要统计专业知识、专门工具和大量时间。我们提出了一个基于METR Task Standard v0.3.0的开放评估任务，测试前沿AI代理是否能自主执行结构化临床AI安全审计。给定一个预训练的临床预测模型、一个患者数据集和书面指令，每个代理必须从伪代码实现四种攻击，计算涵盖FGSM鲁棒性、成员推理抵抗力、期望校准误差和边界攻击抵抗力的安全态势评分，并在Docker容器中仅使用bash接口（无脚手架代码）编写结构化JSON报告。六个变体涵盖威斯康星诊断乳腺癌和MIMIC-IV ICU死亡率数据集，跨三种模型架构，防御强度递增，参考分数范围为55.60至90.41。我们在三个前沿模型上进行了54次评估，每个变体三次运行。Claude Sonnet 4.6和GPT-4.1完成了所有18次运行并获得完美评估分数。GPT-4o完成了61%的运行，每次运行的token数约为Claude的五倍，尽管提供商的token化方式不同。总API成本：GPT-4.1为8美元，Claude Sonnet 4.6为12美元，GPT-4o为27美元。GPT-4o的失败包括过早终止会话、汇总错误和空提交文件。该任务、评分基础设施和威斯康星乳腺癌资产已公开发布；MIMIC-IV变体需要单独访问PhysioNet。

## Abstract
Clinical AI models can expose patients to harm when adversarial vulnerabilities go undetected, yet formal security auditing requires statistical expertise, specialized tools, and significant time. We present an open evaluation task, built on METR Task Standard v0.3.0, that tests whether frontier AI agents can autonomously implement a structured clinical AI security audit. Given a pre-trained clinical prediction model, a patient dataset, and written instructions, each agent must implement four attacks from pseudocode, compute a Security Posture Score covering FGSM robustness, membership inference resistance, expected calibration error, and boundary attack resistance, and write a structured JSON report in a Docker container using only a bash interface and no scaffolding code. Six variants span the Wisconsin Diagnostic Breast Cancer and MIMIC-IV ICU mortality datasets across three model architectures with increasing defense strength, with reference scores from 55.60 to 90.41. We ran 54 evaluations across three frontier models, with three runs per variant. Claude Sonnet 4.6 and GPT-4.1 completed all 18 runs and received perfect evaluator scores. GPT-4o completed 61 percent of runs and used about five times the per-run token count of Claude, although provider tokenization differs. Total API costs were 8 US dollars for GPT-4.1, 12 US dollars for Claude Sonnet 4.6, and 27 US dollars for GPT-4o. GPT-4o failures involved premature session termination, an aggregation error, and an empty submission file. The task, scoring infrastructure, and Wisconsin Breast Cancer assets are publicly released; MIMIC-IV variants require separate PhysioNet access.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：临床AI模型（如预测住院死亡率、药物剂量推荐等）在实际部署前往往缺乏正式的对抗性安全审计。传统审计需要统计专业知识、专用软件和大量时间，而大多数临床部署团队至少缺乏其中一项。这导致模型可能被携带恶意输入的攻击者操纵，造成患者伤害。
- **整体含义**：前沿AI代理（如Claude、GPT-4等）具备多步推理和工具使用能力，理论上可以自主阅读审计规范、编写攻击代码并生成结构化报告，从而弥补临床安全审计的能力缺口。本文旨在通过一个真实、可复现的任务，实证评估前沿AI代理能否胜任这一角色，并比较不同模型的性能差异，为自动化安全审计提供基准。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

- **核心思想**：基于METR Task Standard v0.3.0设计一个封闭式任务，要求AI代理在Docker容器内仅通过bash工具，从零实现四种对抗性攻击，计算复合安全评分并输出JSON格式报告。无任何脚手架代码，代理必须自主编写Python、安装依赖、调试并输出结果。
- **安全态势评分（Security Posture Score, SPS）**：加权复合分数，范围为0~100，越高表示模型越安全。
  \[
  \text{SPS} = (0.35 \cdot s_{\text{fgsm}} + 0.25 \cdot s_{\text{mi}} + 0.20 \cdot s_{\text{ece}} + 0.20 \cdot s_{\text{ba}}) \times 100
  \]
- **四种攻击组件**（每个组件得分∈[0,1]）：
  - **FGSM鲁棒性（s_fgsm，权重35%）**：用逻辑回归代理模型拟合梯度，对100个测试点施加符号扰动（epsilon=0.15 for WDBC，0.05 for MIMIC-IV），计算AUROC下降值，公式 \( s_{\text{fgsm}} = \max(0, 1 - \Delta\text{AUROC}) \)。
  - **成员推断抵抗力（s_mi，权重25%）**：训练4个随机森林影子模型，构建攻击分类器，查询目标模型在训练成员和非成员上的输出，计算攻击准确率a_mi，然后映射为 \( s_{\text{mi}} = \max(0, \min(1, 1 - 2(a_{\text{mi}} - 0.5))) \)。
  - **校准ECE（s_ece，权重20%）**：使用10个均匀宽度概率箱计算期望校准误差，公式 \( s_{\text{ece}} = \max(0, 1 - 12 \cdot \text{ECE}) \)。
  - **边界攻击抵抗力（s_ba，权重20%）**：从50个正确分类的测试点开始，向最近反类点迭代步进（步长0.05，最多100步），取平均步数/100作为得分。
- **任务接口**：Docker容器内仅提供bash工具，最多40轮调用。模型文件和数据集预置在 `/home/agent/`，指令文件描述上述公式和伪代码，代理需自己编写Python代码并输出到 `submission.txt`。
- **评分器**：比较提交值与参考值，SPS容忍度±5分，各组件容忍度±0.10。评分公式：得分 = 0.60 × 1[|SPS误差|≤5] + 0.40 × (1/4)∑组件通过标志。
