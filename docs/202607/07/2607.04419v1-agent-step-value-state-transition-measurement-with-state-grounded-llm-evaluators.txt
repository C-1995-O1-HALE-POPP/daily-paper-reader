## Abstract 

Most agent evaluations collapse a multi-step trace into a final answer, a success flag, or a trajectory-level score. These aggregates obscure the diagnostic question developers need most: which action changed the state in a useful direction? We introduce Agent Step Value (ASV), a state-transition measurement framework that scores each observed action by the change it induces in a state-grounded evaluator’s distribution over fixed candidate outcomes. ASV renders redacted before/after state projections, uses a stateless LLM evaluator to assign candidate log scores, and reports both gold-free belief diagnostics and offline oracle validation metrics. A label-free rationale pass separates evaluator deliberation from one-token option scoring, preserving candidate likelihoods while exposing leakage and floor-score events. On 100 reviewed open-QA evidence-seeking tasks with live PubMed retrieval, a partially live DeepSeek actor, and DeepSeek log-probability scoring, ASV evaluates 1,100 steps and 2,200 states. Under the fixed-layout rationale-conditioned protocol, mean gold-margin gain is -2.335 (trajectory-bootstrap 95% CI [-3.395, -1.272]), entropy movement is 0.000, and mean Bayesian surprise is 2.693. ASV therefore localizes constructive and destructive belief pivots that final-answer scores and entropy-only step metrics miss. We release the standalone ASV Eval toolkit. 

## 1 Introduction 

Language-model agents increasingly act through long traces: they classify requests, plan, retrieve evidence, call tools, draft answers, audit intermediate products, and revise. A final score is useful for ranking systems, but it is a weak debugging signal. If a run fails, the final answer rarely tells us whether retrieval helped, whether synthesis erased useful evidence, or whether a late audit corrected or amplified an error. Evaluation therefore needs a transition-level object. We study this object through Agent Step Value (ASV). Given an observed transition (xi−1, a i, x i),ASV asks how a specified evaluator’s belief over candidate task outcomes changes from the before-state xi−1 to the after-state xi. The actor can be a ReAct-style loop [ 1 ], a DSPy-like stateless module graph [ 2], or a domain workflow with structured traces. ASV only requires that the workflow export bounded state projections and a fixed candidate set for the task. The measurement interface is narrow by design. A state-grounded LLM evaluator scores every candidate in every state and stores raw log scores. This gives more structure than a judge label and more measurability than a free-form critique. ASV then reports gold-free diagnostics, such as entropy movement and Bayesian surprise, and offline oracle validation metrics, such as gold-margin

Evaluator deliberation creates the main measurement issue. If a judge is asked to reason and then answer, the first-token log probabilities measure the beginning of the reasoning string instead of the candidate choice. First, it formalizes step value as evaluator-defined belief movement over observed state transitions. Second, it gives a deliberative LLM-evaluator protocol that preserves candidate log scores by separating label-free rationale generation from one-token option scoring. Third, it shows how entropy movement, Bayesian surprise, and gold-margin gain play distinct diagnostic roles. Fourth, it releases a standalone ASV Eval toolkit and reports a 100-question study in which entropy-only step value is vacuous in near-one-hot regimes while Bayesian surprise and gold-margin gain recover pivot magnitude and direction. Under the tested rationale-conditioned protocol, the average gold-margin movement is negative even though some individual pivots are strongly constructive. Entropy movement is zero for every step, so an entropy-only ASV would miss the transitions that drive the result. Figure 1 summarizes the measurement flow. 

## 2 Related work 

ASV sits between agent benchmarking, process supervision, and LLM-as-judge evaluation. Agent systems such as ReAct, MRKL, Toolformer, ToolLLM, and DSPy show that language-model applica-tions can be built as tool-using or modular programs [ 1, 5– 7, 2]. Benchmarks such as AgentBench, WebArena, Mind2Web, and SWE-bench evaluate increasingly realistic agent behavior at the system level [ 8– 11 ]. ASV is complementary to these benchmarks: when a workflow or benchmark can export before/after states, ASV turns the trace into a sequence of measurable belief updates. Process supervision and step-level agent evaluation provide the closest methodological motivation. PRM800K demonstrates the value of labels on intermediate reasoning steps [ 12 ], while Agent-as-a-Judge and AgentEval use agentic evaluators and structured step feedback for richer diagnosis [ 13 , 14 ]. ASV uses a smaller and more portable contract: a transition, a candidate set, and candidate log scores. This makes it usable when human process labels, trained reward models, or explicit dependency graphs are unavailable. 2LLM-as-judge work shows that language models can approximate human preferences, while remain-ing sensitive to prompt format and option order [ 15 –17 ]. ASV treats the evaluator as a measurement instrument whose protocol must be specified and audited. It records raw scores, floor-score events, rationale leakage flags, and label mappings. The evidence-seeking setting also connects to retrieval-grounded evaluation in FEVER, KILT, PubMedQA, and BioASQ [ 18 – 21 ], although ASV is not tied to biomedical tasks. Finally, we use entropy and Bayesian surprise as replay diagnostics; this complements calibration and adaptive fine-tuning views of entropy [ 22 ] and classical accounts of surprise as belief change [23]. 

## 3 Agent step value 

ASV is a replay measurement procedure. It takes an observed trace, a fixed candidate set, and an evaluator protocol, then computes how evaluator belief changes at each transition. The method separates three quantities that are often conflated: gold-free belief movement, offline target-directed movement, and evaluator quality flags. This separation is what lets ASV diagnose a trace without treating the evaluator as ground truth. 

3.1 Problem setup 

Let a task be defined by an input q and a finite candidate set C = {c1, . . . , c K }. Each candidate has a stable identifier u(c) ∈ U that is independent of any visible option label. A trajectory is an ordered sequence 

T = (( x0, a 1, x 1), . . . , (xn−1, a n, x n)) , (1) where ai is the observed action and xi−1, x i are serialized states before and after the action. ASV is agnostic to how the actor produced the transition. It only assumes that a state projection 

zx = Φ( x, q, C ) (2) can be rendered for the evaluator. The projection removes physical option labels, gold labels, success flags, answer keys, final scores, raw provider responses, request identifiers, timestamps, trace identifiers, and secrets. Candidate texts may be included for open-QA tasks because identifiers such as answer-a have no semantics. The gold candidate g is retained only for offline analysis. ASV measures the change in a specified evaluator’s state-conditioned belief over C. It does not estimate intrinsic causal utility; that would require interventions on the actor or environment. The replay quantity is narrower: under a fixed projection Φ and evaluator protocol Eθ , did the observed action move evaluator belief toward, away from, or across candidate outcomes?

## 4 Experiment 

We ran a 100-question open-QA experiment to test whether ASV produces interpretable step-level measurements on realistic evidence-seeking traces. The tasks span intervention, risk, mechanism, diagnostic, and insufficient-evidence questions. Each task has four reviewed candidates: three concrete answers and one none-of-the-above . The reviewed target distribution is intentionally not balanced across physical candidate identifiers because candidate identifiers are stable semantic handles, not labels shown to the evaluator. Trajectories were collected from an evidence-retrieval workflow using PubMed through NCBI Entrez. The actor used DeepSeek where configured: classification used LLM mode for 97 of 100 trajectories, planning for 67, synthesis for 52, verification for 95, and revision for 50; the remaining substeps used deterministic or fallback paths recorded in the trace metadata. We then reran the ASV evaluator with DeepSeek deepseek-chat , temperature zero, one generated token for the scored pass, log probabilities enabled, top-log-probabilities requested for option labels, a 128-token label-free rationale budget, warning-mode leakage policy, and fixed physical option layout. The run completed all 100 trajectories, yielding 1,100 agent steps and 2,200 evaluated before/after states. Table 1 reports the main result. Mean gold-margin gain is -2.335, with a trajectory-bootstrap 95% confidence interval of [-3.395, -1.272]. Leave-one-trajectory-out means range from -2.467 to -2.180, so no single trajectory explains the sign. The negative mean remains informative because the evaluated stack struggles even when non-answerability is available as a candidate outcome. 65 Discussion 

The experiment supports the main measurement claim. If an agent trace can be represented as before/after states and a candidate set can be frozen, a stateless evaluator can turn each action into a measurable belief update. ASV does not estimate causal utility. It gives a local replay diagnostic: which observed transitions moved the evaluator toward the reviewed target, which moved it away, and which changed belief mass without improving the oracle margin. The negative mean gold-margin gain is evidence about this measured stack, not a verdict on agents as a class. The tested rationale-conditioned protocol exposes destructive pivots in the open-QA traces. The actor was partly live and partly fallback, the state projection compressed evidence, and the evaluator relied on a compact rationale buffer. ASV makes these surfaces visible: fallback/floor counts, leakage flags, candidate coverage, raw score vectors, entropy, surprise, and margin movement are all retained for audit. The entropy result gives the sharpest methodological lesson. Entropy movement is zero across 1,100 steps because the evaluator often behaves like a near-one-hot classifier before and after a transition. Yet Bayesian surprise and gold-margin gain reveal large belief flips. In this regime, an information-theoretic diagnostic must measure distributional movement alongside uncertainty reduction. Bayesian surprise catches the pivot; gold-margin gain gives the pivot a direction relative to the reviewed target. The label-free rationale buffer should also be interpreted carefully. It makes evaluator deliberation explicit while preserving a one-token likelihood measurement, but it is not guaranteed to improve scores. A rationale can compress useful evidence, omit decisive evidence, or anchor the scorer on a misleading comparison. The claim is narrower: ASV makes deliberation measurable and auditable. It does not assert that deliberative judging is always superior to direct one-token scoring. 

## 6 Limitations and broader impact 

This paper reports one 100-question open-QA experiment with limited scope. The tasks use reviewed candidate sets, live PubMed retrieval, a partially live DeepSeek actor, and one DeepSeek evaluator protocol. The results should not be generalized to other domains, candidate-construction procedures, evaluators, or fully live actors without rerunning the measurement. The bootstrap interval and leave-one-trajectory-out range quantify sensitivity within this dataset; they do not replace external replication. The evaluator is an LLM and inherits model-specific biases, including option-order sensitivity. ASV exposes these risks by storing raw scores, separating rationale generation from one-token scoring, logging leakage and floor-score events, and supporting permutation audits, but these logs do not make the evaluator objective. Candidate construction is also part of the measurement: overlapping candidates, missing targets, or artifacts correlated with the target can distort margin scores. The broader impact is mainly methodological. ASV can help researchers and developers identify which parts of a long agent trace improve evidence-grounded correctness and which parts degrade it. The corresponding risk is overtrust: a numeric step value can look more objective than the evaluator protocol that produced it. In high-stakes settings, ASV should remain a debugging and evaluation tool, reported with source evidence, evaluator details, candidate specifications, and robustness diagnostics. 

## 7 Conclusion 

ASV reframes agent evaluation as state-transition measurement. A stateless, state-grounded LLM evaluator scores before and after states over a fixed candidate set; raw candidate log scores then yield entropy movement, Bayesian surprise, and offline oracle margin gains. In a 100-question open-QA experiment with 1,100 steps, ASV exposes a stable negative mean gold-margin movement under the tested rationale-conditioned protocol and shows that entropy alone is blind to large one-hot belief pivots. The released toolkit gives a compact way to localize where an agent trace changes evaluator belief and whether that change aligns with a reviewed target. 7Code and data availability 

The standalone ASV Eval toolkit is publicly available at https://github.com/andyzpb/ asv-eval . The repository contains the standard ASV JSONL schema, metric implementation, evaluator protocol, command-line interface, documentation, tests, and runnable offline examples. The raw 100-question trajectory artifacts and provider log-probability responses are not included in the public release because they require credential scrubbing, license review, and artifact redaction. Exact reproduction of the reported numbers therefore requires access to the retained JSONL traces and provider responses, while the released toolkit supports independent reimplementation and new ASV evaluations.

## A Reproducibility details 

The implementation stores trajectories as JSONL records containing task metadata, candidate spec-ifications, and ordered state transitions. Each transition contains an action name, a before-state projection, an after-state projection, and optional cost and trace metadata. Evaluator outputs are stored separately as candidate log scores, normalized beliefs, rationale quality flags, permutation identifiers, and metric records. Gold candidate identifiers, success labels, final scores, and physical option labels are never rendered into the label-free state projection. This separation makes it possible to rerun evaluator protocols from preserved trajectories without rerunning the actor. 9The 100-question open-QA experiment used four candidates per task, including exactly one 

none-of-the-above . ASV restricts scoring to a small option set because first-token top-log-probability APIs expose only a bounded set of candidate tokens. The source trajectories were collected with PubMed retrieval through NCBI Entrez. No model training was performed. The evaluator used deepseek-chat , a 128-token label-free rationale pass, temperature zero, one gener-ated token for the scored pass, log probabilities enabled, and top-log-probabilities requested for the option labels. The main reported estimate uses a fixed option layout; label-permutation scoring is implemented as a robustness audit but is not used in the main estimate reported here. 

## B Evaluator prompt skeleton 

ASV uses two evaluator prompts. The first prompt generates a compact rationale buffer without physical labels. The implementation renders the task question, the redacted state projection, and a candidate manifest using stable candidate identifiers and candidate texts. 

> You are evaluating an agent state. Generate a compact label-free rationale buffer. Use stable candidate identifiers only. Never mention physical option labels such as A/B/C/D, gold labels, answer keys, success flags, or scores. Compare the evidence against each candidate identifier and separate supporting evidence, contradicting evidence, missing evidence, and unresolved ambiguity. Do not output a final option label.

The second prompt appends the physical label mapping only at the tail and scores one token.  

> [Redacted state projection.] [Candidate manifest with candidate identifiers and texts.] [Label-free rationale buffer.] Current physical option mapping: A = candidate_id: [id]. B = candidate_id: [id]. C = candidate_id: [id]. D = candidate_id: [id]. Output exactly one uppercase option label.

The scored call is read from the log probabilities of the first generated option token. Label tokens are normalized by stripping whitespace and simple punctuation, so variants such as A, leading-space A,newline-prefixed A, and A. map to the same option; when multiple returned token variants map to the same option, their scores are combined with log-sum-exp. The scored prompt is not allowed to emit free-form reasoning before the option label because that would make first-token log probabilities measure reasoning preambles. 10
