## 摘要
当对抗性漏洞未被检测时，临床AI模型可能使患者受到伤害，而正式的安全审计需要统计专业知识、专业工具和大量时间。我们提出了一个基于METR任务标准v0.3.0的开放评估任务，测试前沿AI智能体是否能自主实施结构化的临床AI安全审计。给定一个预训练的临床预测模型、一个患者数据集和书面指令，每个智能体必须从伪代码实现四种攻击，计算涵盖FGSM鲁棒性、成员推理抵抗性、预期校准误差和边界攻击抵抗性的安全态势得分，并在Docker容器中仅使用bash界面且无脚手架代码编写结构化的JSON报告。六个变体涵盖威斯康星诊断乳腺癌和MIMIC-IV ICU死亡率数据集，使用三种模型架构且防御强度递增，参考得分从55.60到90.41。我们在三个前沿模型上进行了54次评估，每个变体运行三次。Claude Sonnet 4.6和GPT-4.1完成了所有18次运行并获得了完美的评估得分。GPT-4o完成了61%的运行，每次运行的token数量约为Claude的五倍，尽管提供商的token化方式不同。GPT-4.1的总API成本为8美元，Claude Sonnet 4.6为12美元，GPT-4o为27美元。GPT-4o的失败包括过早的会话终止、聚合错误和空提交文件。该任务、评分基础设施和威斯康星乳腺癌资产已公开发布；MIMIC-IV变体需要单独的PhysioNet访问权限。

## Abstract
Clinical AI models can expose patients to harm when adversarial vulnerabilities go undetected, yet formal security auditing requires statistical expertise, specialized tools, and significant time. We present an open evaluation task, built on METR Task Standard v0.3.0, that tests whether frontier AI agents can autonomously implement a structured clinical AI security audit. Given a pre-trained clinical prediction model, a patient dataset, and written instructions, each agent must implement four attacks from pseudocode, compute a Security Posture Score covering FGSM robustness, membership inference resistance, expected calibration error, and boundary attack resistance, and write a structured JSON report in a Docker container using only a bash interface and no scaffolding code. Six variants span the Wisconsin Diagnostic Breast Cancer and MIMIC-IV ICU mortality datasets across three model architectures with increasing defense strength, with reference scores from 55.60 to 90.41. We ran 54 evaluations across three frontier models, with three runs per variant. Claude Sonnet 4.6 and GPT-4.1 completed all 18 runs and received perfect evaluator scores. GPT-4o completed 61 percent of runs and used about five times the per-run token count of Claude, although provider tokenization differs. Total API costs were 8 US dollars for GPT-4.1, 12 US dollars for Claude Sonnet 4.6, and 27 US dollars for GPT-4o. GPT-4o failures involved premature session termination, an aggregation error, and an empty submission file. The task, scoring infrastructure, and Wisconsin Breast Cancer assets are publicly released; MIMIC-IV variants require separate PhysioNet access.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 核心思想
- 基于METR任务标准v0.3.0，设计一个封闭式任务：AI代理在Docker容器内，仅通过bash工具接口，根据书面指令（伪代码）实现四种对抗性攻击算法，计算安全态势评分（Security Posture Score, SPS），并输出结构化的JSON报告。无任何脚手架代码提供，代理需自行编写Python代码、安装库、执行并聚合结果。

### 2.2 关键技术细节
- **安全态势评分（SPS）**：SPS = 0.35·s_fgsm + 0.25·s_mi + 0.20·s_ece + 0.20·s_ba，结果乘以100，范围0-100。分值越高表示模型越安全。
  - **FGSM鲁棒性（s_fgsm）**：拟合替代逻辑回归（30%训练数据子样本），生成对抗样本（ε=0.15用于WDBC, ε=0.05用于MIMIC-IV），计算AUROC下降量。s_fgsm = max(0, 1 - ΔAUROC)。
  - **成员推理抵抗性（s_mi）**：训练4个随机森林影子模型，训练攻击二分类器，对目标模型查询最多200个训练成员和200个非成员，计算攻击准确率a_mi。s_mi = max(0, min(1, 1 - 2(a_mi - 0.5)))。完美攻击者得0，随机猜测得1。
  - **校准预期校准误差（s_ece）**：10个等宽概率箱计算ECE。s_ece = max(0, 1 - 12·ECE)。ECE为1/12时得0，完美校准时得1。
  - **边界攻击抵抗性（s_ba）**：从50个随机测试点出发，向最近异类点迭代步进（步长0.05，最多100步），记录步数平均值。s_ba = mean(steps)/100。
- **任务接口**：Docker容器（2 vCPU, 4 GiB RAM），代理仅有一个bash工具，最多40次bash调用。任务指令文件包含SPS公式、每个攻击的伪代码、输出格式要求。无网络访问。
- **评估计分**：METR评分器比较提交值与锁定参考值。SPS容忍度±5分，每个组件容忍度±0.10。得分 = 0.60·1[|ŝ-s*|≤5] + 0.40·(1/4)∑1[|ĉ-c*|≤0.10]。满分1.0。

## 6. 论文的主要结论与发现

- **主要结论**：前沿AI代理能够以高保真度自主执行结构化的临床AI安全审计。Claude Sonnet 4.6和GPT-4.1在所有18次运行中均获得完美评分（1.0），无任何失败。GPT-4o完成61%（11/18），其余7次失败。
- **关键发现**：
  1. **能力差距**：同为“前沿”模型，性能差异显著。GPT-4o的39%失败率对部署决策有实际影响。

  3. **GPT-4o三类失败模式**：
     - **过早会话终止**（5次）：在写入文件前关闭会话，即使已计算正确中间值。
     - **算术聚合错误**（1次）：手动计算SPS公式时权重应用错误，各组件正确但总分偏差-22分。
     - **空提交文件**（1次）：文件存在但无内容（可能写入被截断）。
  4. **成本差异**：GPT-4o总API成本（$27）是Claude（$12）的两倍多，且失败运行消耗了44.6%的输入token却无有效结果。
  5. **临床部署意义**：一个机构只需scikit-learn兼容模型、数据集和API访问，即可在约25分钟内完成六变体审计，输出结构化JSON报告，远快于人工审计数周。但需针对新模型调整变体，且MIMIC-IV受限于PhysioNet凭证。

- **任务设计精良**：将真实临床安全审计需求转化为可自动验证的METR任务，涵盖四种不同攻击维度和加权评分机制，容忍度校准合理（±5 SPS，±0.10组件），兼顾实现差异和错误捕捉。

## 8. 不足与局限

- **模型数量有限**：仅评估三个模型，无法泛化到所有商业化代理。其他前沿模型（如Gemini、Llama 3等）未被测试。
- **数据集范围窄**：WDBC数据集规模小（569样本）、特征干净；MIMIC-IV虽更真实但仅为ICU死亡率预测。真实临床环境包含缺失值、噪声、高维数据和复杂标签，未覆盖。
- **任务透明度高**：指令提供了伪代码和参数，测试的是实现和执行能力而非攻击方法论推导。未来可设计“无伪代码”版本。
- **安全覆盖不全**：SPS仅覆盖四种攻击，未包含数据投毒、模型提取、跨架构迁移攻击、对深度神经网络的评估等威胁类型。
- **评分容忍度缺陷**：±0.10的组件容忍度对接近零的组件过于宽松（例如WDBC基线FGSM参考值0.004，提交0.104可通过检查，但实际高了24倍）。虽未影响本次评估，但在高安全场景下需修正。
- **浅层分类器局限**：所有六种架构均为逻辑回归、随机森林、XGBoost等浅层模型，未覆盖CNN、Transformer等深度网络（临床影像领域常用）。任务扩展至深度模型时需调整梯度替代、步长等参数。
- **MIMIC-IV可复现性受限**：MIMIC-IV变体需要独立PhysioNet凭证，无法直接复现，降低外部分验便利性。
- **运行日志缺失**：评估日志仅记录聚合得分、回合数和token用量，无逐回合bash痕迹，因此失败模式分析基于推断而非直接观察，准确性受限。
- **小样本统计不确定性**：每变体仅3次运行，GPT-4o的完成率置信区间较宽，限制了精确点估计。
